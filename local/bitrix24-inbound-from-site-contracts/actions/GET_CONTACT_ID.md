# ACTION: `GET_CONTACT_ID`

Поиск контакта CRM по email и/или телефону с разрешением неоднозначности через телефон.

## Обязательные поля тела запроса

| Поле | Тип | Описание |
|------|-----|----------|
| `ACTION` | string | `GET_CONTACT_ID` |

## Идентификация контакта

Должен быть передан хотя бы один из:

| Поле | Описание |
|------|----------|
| `EMAIL` | После trim непустая строка |
| `PHONE` | После trim непустая строка |

Порядок в коде (`InboundEndpoint::handleGetContactId`):

1. Если задан email — выбираются контакты с точным совпадением по email в CRM.
   - Ровно один контакт → успех.
   - Несколько контактов и задан телефон — среди кандидатов ищется тот, у кого телефон совпадает (нормализация цифр как в обработчике).
2. Если контакт ещё не найден и задан телефон — поиск по телефону с вариантами номера (`phoneVariants`).
   - Успех только при **единственном** совпадении.

## Успешный ответ

- `success`: `1`
- `data.ID`: числовой идентификатор контакта в CRM

## Ошибки

- Контакт не найден: `success`: `0`, `error`: сообщение вида «Contact not found», `data`: пустой массив.
