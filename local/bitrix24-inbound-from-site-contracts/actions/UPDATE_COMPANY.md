# ACTION: `UPDATE_COMPANY`

Обновление компании CRM по связке с элементом инфоблока на сайте: ищется компания по **ID элемента на сайте**, затем применяются переданные поля.

Реализация: `InboundEndpoint::handleUpdateCompany`.

## Обязательное условие: идентификатор элемента на сайте

Должен быть указан **ровно один** из источников (после приведения к строке и `trim`), и значение **не может быть пустым и не может быть `'0'`** — это соответствует требованию хранить реальный положительный ID элемента, а не заглушку.

| Источник | Описание |
|----------|----------|
| UF канонический | Ключ из карты `company.site_element_id` → **`UF_CRM_1774915439581`** |
| UF legacy | Ключ `company.site_element_id_legacy_alias` → **`UF_CRM_3804624439373`** (совместимость со старыми обработчиками и зеркалированием исходящего `CompanySync`) |
| `SITE_ELEMENT_ID` | Общее скалярное поле-транспорт то же значение |

Поиск компании выполняется по **обоим** UF: если исторически заполнен только legacy-UF, компания всё равно находится.

Уникальность: должна совпасть **ровно одна** компания по указанному значению среди записей с любым из двух UF. Иначе — ошибка с массивом `matched_ids`.

## Поля данных (опционально, по ключам запроса)

| Ключ запроса | Куда пишется в CRM |
|----------------|-------------------|
| `LEGAN_ENTITY_NAME` | `TITLE` |
| `LEGAN_ENTITY_CITY` | UF города (`company.city` → смотри `uf_mapping.php`) |
| Канонический UF или legacy UF элемента сайта | При наличии любого из ключей в теле запроса оба UF (**канонический и legacy**) получают одно и то же значение идентификатора элемента |
| `LEGAN_ENTITY_PHONE` | Мультполе `PHONE` (WORK), если не пусто |
| `LEGAN_ENTITY_EMAIL` | Мультполе `EMAIL` (WORK), если не пусто |
| `LEGAN_ENTITY_WWW` | Мультполе `WEB` (WORK), если не пусто |

Если ни одно поле для обновления не собралось в `$fields`, запись CRM не меняется; ответ при этом всё равно может быть `success`: `1` (нет ошибки выполнения). Для интеграций имеет смысл всегда передавать хотя бы одно изменяемое поле, когда нужна реальная запись в CRM.

## Связь с исходящим каналом

Исходящая синхронизация компании в сторону сайта (`CompanySync`) отправляет оба UF и полный набор `OS_*` / `LEGAN_*`. Чтобы не получить рассинхрон («создали компанию в CRM без реального ID элемента на сайте»), при создании через `CRM_METHOD` / CRM UI нужно явно заполнить канонический UF и не опираться на ноль как на валидный ID.

## Ответ

- Успех: `success`: `1`, `data.ID` — Bitrix ID компании, `data.matched_by` — канонический код UF для связи.
